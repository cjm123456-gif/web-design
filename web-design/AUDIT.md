# 页面实现与质量审计

## 1. 参考图结构审计

参考图中可识别的页面模块：

1. 黑色固定导航与双 CTA。
2. 紫色发光地平线 Hero、大标题与按钮。
3. 悬浮数据/信任条。
4. 六张服务能力卡片，其中一张使用高亮紫色。
5. 整段紫色优势区与四张编号卡片。
6. 深色项目成果区与项目卡片。
7. 底部延伸内容区域的开头。

本网站的对应模块：

1. 浮动材质导航、中性智能体标识与展示入口。
2. 紫色地平线光弧 Hero、中文业务主张与双 CTA。
3. 四项服务能力概览与五节点能力脉冲轨道。
4. 智能体概览与高校智能体能力连接图。
5. 六类智能体能力卡片，其中课程教学智能助教为高亮卡。
6. 紫色应用价值区与四张编号卡片。
7. 不等大 Bento 成品展示区。
8. 建设路径、探索更多场景与中性页尾。

## 2. 对齐项

- 深黑背景、紫色主强调色、白色标题和灰紫正文层级与参考图一致。
- Hero 保留大型发光弧、中心式大标题、双按钮和悬浮信任条。
- 能力区保留三列六卡结构和单张紫色重点卡。
- 优势区保留整段紫色背景、四列编号卡和细线描边。
- 成果区延续深色描边卡片，但使用不等大 Bento 布局强化层级。
- 桌面端保持宽容器和较大章节间距；移动端正文卡片折叠为单列，成品区在独立容器内横向浏览。
- L2 动效覆盖短时标题入场、滚动渐显、导航材质变化、轻视差、卡片聚光、即时按压与球面流体背景；常驻循环仅保留这一处首屏签名效果。

## 3. 有意差异

- 未复制参考图中的 Fintech 标识、英文文案或数据。
- 页面使用中性抽象智能体标识，不出现任何企业或地区属性。
- 未添加无法核实的项目、成果数字、评价或资质声明。
- 成品区当前是明确标注的界面示意和资料占位，不冒充真实案例。
- 参考图为静态截图，无法确认原站滚动动效；因此按用户确认的 L2 档位独立设计，没有推测或复刻未知动效。
- 未加入 WebGL、滚动劫持或全局自定义光标，以保证高校商务场景的稳重感和滚动性能。

- [x] Hero H1：约 560ms 逐行 clip reveal + 关键词一次性渐变。
- [x] Section H2：IntersectionObserver 驱动的 ScrollFloat 风格进入。
- [x] Body/Label：eyebrow 与段落渐显。
- [x] 元素级：取消会移动命中区域的 CTA 磁吸，改为 pointer-down 即时按压与平滑回位。
- [x] 交互构件：Spotlight 能力卡和 Bento 成品卡。
- [x] 氛围背景：球体内三层 CSS 流体以 18/24/30 秒错速漂移，只动画 transform；外圈紫色条带使用双边界 SVG 正弦路径持续形变。
- [x] 关系示意图：精细指针进入后，网格、轨道、四个节点与中心核心按不同深度直接跟随；离开后平滑回中。
- [x] 首屏离开视口或页面失焦时，CSS 流体和 SVG 波面均暂停；移动端条带限至 30fps，减少动态偏好下冻结为静态表面。
- [x] `pointermove` 均通过 `requestAnimationFrame` 节流。
